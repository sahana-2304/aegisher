AegisHer ML Model Documentation
Last updated: March 9, 2026

1) Purpose
This model predicts location-level safety risk as a binary probability:
- 0.0 -> low risk
- 1.0 -> high risk

The ML output is used in two ways:
- Direct model testing via dev endpoints.
- Hybrid production risk scoring where AI score is combined with community intelligence.


2) Source Dataset
File:
- backend/data/tamil_nadu_women_safety_dataset.csv

Observed structure:
- Row count: 80,000 data rows (plus header)
- Target column: risk_zone (0 or 1)
- Main columns:
  - latitude
  - longitude
  - city
  - hour
  - day_type
  - lighting_score
  - crowd_density
  - crime_density_norm
  - police_distance_km
  - cctv_presence
  - land_use_type
  - past_incident_count
  - risk_zone


3) Training Pipeline
Implementation file:
- backend/services/risk_model_trainer.py

Training entrypoints:
- CLI script: backend/scripts/train_risk_model.py
- API endpoint (dev only): POST /api/risk/model/train

Pipeline steps:
1. Load CSV with pandas.
2. Validate required columns:
   latitude, longitude, hour, day_type, lighting_score, crowd_density,
   crime_density_norm, police_distance_km, cctv_presence, past_incident_count, risk_zone.
3. Feature engineering:
   - day_type_flag = 1 if weekend else 0
   - crime_density = crime_density_norm normalized to [0,1] if values are > 1
4. Convert all feature columns to numeric.
5. Drop rows with missing feature/target values.
6. Optional downsampling:
   - Controlled by env: RISK_TRAIN_SAMPLE_SIZE
   - If > 0 and dataset is larger, random sample is used.
7. Split data:
   - train_test_split(test_size=0.2, stratify=y, random_state=42)
8. Train model:
   - HistGradientBoostingClassifier with:
     - learning_rate=0.08
     - max_iter=260
     - max_depth=8
     - min_samples_leaf=20
     - random_state=42
9. Evaluate:
   - accuracy
   - precision
   - recall
   - f1
   - roc_auc
10. Save artifact with joblib to backend/models/risk_model.pkl.


4) Model Artifact Format
Saved object is a dictionary with:
- model: trained sklearn model
- feature_columns: ordered feature list
- feature_medians: median value per feature (from X_train)
- target_column: "risk_zone"
- trained_at: UTC ISO timestamp
- dataset_path: path used for training
- samples_used: effective row count used for training


5) Runtime Inference Engine
Implementation file:
- backend/services/risk_engine.py

Model loading behavior:
- Uses RISK_MODEL_PATH if provided.
- Else defaults to backend/models/risk_model.pkl.
- Relative paths are resolved against backend directory.

Prediction behavior:
1. Build feature map from latitude/longitude plus optional context.
2. Fill defaults for missing context:
   - hour from current UTC time
   - day_type inferred from weekday/weekend
   - lighting/crowd/crime/police/cctv/past incidents from fallback heuristics
3. Clip values into safe ranges.
4. If model is loaded:
   - predict_proba (if available), else predict
5. If model is unavailable or inference fails:
   - fallback to heuristic risk formula.

Output:
- risk_score in [0,1]
- shap_values (feature contribution map used for UI/debug)
- model_source ("artifact" or "heuristic")


6) Hybrid Risk Scoring
Implementation file:
- backend/routers/risk.py

Production risk endpoint:
- POST /api/risk/predict

Formula:
- final_score = (0.7 * ai_score + 0.3 * community_score) * 100

Zone mapping:
- < 35: LOW
- 35 to < 65: MEDIUM
- >= 65: HIGH

Notes:
- ai_score comes from RiskEngine (ML or heuristic fallback).
- community_score comes from CommunityIntelService.


7) Dev-only Model Management Endpoints
Implementation file:
- backend/routers/risk.py

Guard:
- _ensure_dev_mode() blocks these endpoints when APP_ENV is production/prod.

Endpoints:
- GET /api/risk/model/status
  - Returns model_loaded, model_source, model_path, feature_columns
- POST /api/risk/model/train
  - Trains from dataset and reloads model in memory
- POST /api/risk/model/test
  - Runs direct model prediction from supplied feature payload

Test payload supports:
- latitude, longitude
- hour, day_type
- lighting_score, crowd_density, crime_density_norm
- police_distance_km, cctv_presence, past_incident_count


8) Frontend Model Test UI (Dev only)
File:
- frontend/src/screens/ModelTestingScreen.jsx

Capabilities:
- Refresh model status
- Train model from dataset
- Test model with manual feature inputs
- Use current geolocation for latitude/longitude
- Display probability, label, and contributing factors JSON

Route visibility:
- Route /ml-test exists only when import.meta.env.DEV is true.
- It is intentionally not shown in the bottom navigation bar.


9) Environment Variables
Backend:
- APP_ENV=development | production
- RISK_MODEL_PATH=models/risk_model.pkl
- RISK_TRAIN_SAMPLE_SIZE=<int> (optional; 0 means use full dataset)

Frontend:
- VITE_API_URL=http://localhost:8000 (or deployed API URL)

12) Known Constraints
- Model is trained on one regional dataset; generalization outside similar data distributions may be limited.
- Real-time contextual features are partially inferred heuristically unless explicitly supplied.
- Risk output is probability-like and should be interpreted as decision support, not certainty.
